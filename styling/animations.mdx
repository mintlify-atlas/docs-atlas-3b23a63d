## Overview

The I/O Festival 2026 website uses Framer Motion for declarative animations, smooth scroll with Lenis, and custom CSS keyframe animations to create an immersive, fluid user experience.

## Framer Motion Setup

### Installation

Framer Motion is included in the project dependencies:

```json package.json
{
  "dependencies": {
    "framer-motion": "^11.x.x"
  }
}
```

### Basic Usage

Import and use motion components:

```tsx
import { motion } from 'framer-motion';

<motion.div
  initial={{ opacity: 0, y: 30 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.7 }}
>
  Content
</motion.div>
```

## Scroll-Based Animations

### whileInView Animations

Elements animate when they enter the viewport using `whileInView`:

<CodeGroup>

```tsx Fade In from Bottom
<motion.div
  initial={{ opacity: 0, y: 30 }}
  whileInView={{ opacity: 1, y: 0 }}
  viewport={{ once: true, amount: 0, margin: "0px 0px 800px 0px" }}
  transition={{ duration: 0.5 }}
>
  Content appears when scrolled into view
</motion.div>
```

```tsx Fade In from Left
<motion.div
  initial={{ opacity: 0, x: -50 }}
  whileInView={{ opacity: 1, x: 0 }}
  viewport={{ once: true }}
  transition={{ duration: 0.8 }}
>
  Content slides in from left
</motion.div>
```

```tsx Staggered Children
{pillars.map((pillar, index) => (
  <motion.div
    key={index}
    initial={{ opacity: 0, y: 30 }}
    whileInView={{ opacity: 1, y: 0 }}
    viewport={{ once: true }}
    transition={{ duration: 0.5, delay: index * 0.1 }}
  >
    {pillar.title}
  </motion.div>
))}
```

</CodeGroup>

### Viewport Configuration

The `viewport` prop controls when animations trigger:

```tsx
viewport={{ 
  once: true,                    // Only animate once
  amount: 0,                     // Trigger immediately when any part is visible
  margin: "0px 0px 800px 0px"   // Trigger 800px before element enters viewport
}}
```

## Smooth Scroll

### Lenis Integration

Smooth scrolling is implemented using React Lenis:

```tsx SmoothScroll.tsx
import { ReactLenis } from 'lenis/react';

export function SmoothScroll({ children }: { children: React.ReactNode }) {
  return (
    <ReactLenis
      root
      options={{
        lerp: 0.1,          // Smoothness (0-1, lower = smoother)
        duration: 1.2,       // Scroll duration
        smoothWheel: true,   // Enable smooth wheel scrolling
        syncTouch: false,    // Disable on touch devices
      }}
    >
      {children}
    </ReactLenis>
  );
}
```

### Applied in Layout

```tsx layout.tsx
export default function RootLayout({ children }) {
  return (
    <html lang="id">
      <body>
        <SmoothScroll>
          {children}
        </SmoothScroll>
      </body>
    </html>
  );
}
```

<Note>
Smooth scroll is disabled on touch devices (`syncTouch: false`) to maintain native scrolling performance on mobile.
</Note>

## Navbar Animations

### Entrance Animation

The navbar slides down on page load:

```tsx Navbar.tsx
<motion.nav
  initial={{ y: -100, opacity: 0 }}
  animate={{ y: 0, opacity: 1 }}
  transition={{ duration: 0.8, ease: [0.16, 1, 0.3, 1] }}
>
  {/* Navbar content */}
</motion.nav>
```

### Scroll-Based Transform

Navbar morphs on scroll using state-driven classes:

```tsx Navbar.tsx
const [isScrolled, setIsScrolled] = useState(false);

useEffect(() => {
  const handleScroll = () => setIsScrolled(window.scrollY > 50);
  window.addEventListener('scroll', handleScroll, { passive: true });
  return () => window.removeEventListener('scroll', handleScroll);
}, []);

const transitionClass = isScrolled 
  ? 'transition-all duration-700 ease-[cubic-bezier(0.16,1,0.3,1)]' 
  : 'transition-all duration-300 ease-[cubic-bezier(0.7,0,0.84,0)]';

<div className={`
  ${transitionClass}
  ${isScrolled
    ? 'rounded-2xl px-6 py-3 bg-black/30 border border-white/10'
    : 'rounded-none px-12 py-5 bg-black/40 border-b border-white/5'
  }
`}>
```

## Preloader Animation

Phased preloader with timed state transitions:

```tsx Preloader.tsx
const [phase, setPhase] = useState<'logo-in' | 'logo-out' | 'bg-out' | 'done'>('logo-in');

useEffect(() => {
  const t1 = setTimeout(() => setPhase('logo-out'), 400);
  const t2 = setTimeout(() => setPhase('bg-out'), 700);
  const t3 = setTimeout(() => setPhase('done'), 1000);
  return () => { clearTimeout(t1); clearTimeout(t2); clearTimeout(t3); };
}, []);

return (
  <AnimatePresence>
    {phase !== 'done' && (
      <motion.div
        animate={{
          opacity: phase === 'bg-out' ? 0 : 1,
        }}
        transition={{ duration: 0.8, ease: 'easeInOut' }}
      >
        <motion.div
          animate={{
            opacity: phase === 'logo-in' ? 1 : 0,
            scale: phase === 'logo-in' ? 1 : 1.05,
            y: phase === 'logo-in' ? 0 : -10,
          }}
          initial={{ opacity: 0, scale: 0.9, y: 5 }}
          transition={{ duration: 0.8, ease: 'easeInOut' }}
        >
          <Image src="/logo.webp" alt="I/O Festival" />
        </motion.div>
      </motion.div>
    )}
  </AnimatePresence>
);
```

## Custom Easing Functions

### Smooth Ease-Out Curve

```tsx
transition={{ 
  duration: 0.8, 
  ease: [0.16, 1, 0.3, 1]  // cubic-bezier for smooth deceleration
}}
```

### Reverse Ease-In Curve

```tsx
transition={{ 
  duration: 0.3, 
  ease: [0.7, 0, 0.84, 0]  // Sharp acceleration
}}
```

## Performance Optimization

### GPU Acceleration

Use `transform-gpu` and `will-change` for smooth animations:

```tsx
<div className="transform-gpu" style={{ willChange: 'transform, opacity' }}>
  {/* Animated content */}
</div>
```

### Passive Event Listeners

```tsx
window.addEventListener('scroll', handleScroll, { passive: true });
```

### Viewport Margins

Trigger animations early to ensure they're complete before users see them:

```tsx
viewport={{ margin: "0px 0px 800px 0px" }}
```
