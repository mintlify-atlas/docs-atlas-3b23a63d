# Installation

This guide provides comprehensive instructions for setting up the I/O Festival 2026 project on your local development environment.

### System requirements

- **Operating System**: macOS, Windows, or Linux
- **RAM**: 4GB minimum, 8GB+ recommended
- **Disk Space**: 500MB for dependencies
- **Browser**: Modern browser with WebGL 2 support for 3D features

<Note>
  The project uses React 19 and Next.js 16 which have specific Node.js version requirements. Ensure you're using Node.js 18.18 or later.
</Note>

## Installation steps

<Steps>
  <Step title="Verify Node.js installation">
    First, confirm you have the correct Node.js version:
    
    ```bash
    node --version
    ```
    
    You should see `v18.18.0` or higher. If not, download and install the latest LTS version from [nodejs.org](https://nodejs.org/).
    
    Also check npm:
    ```bash
    npm --version
    ```
  </Step>

  <Step title="Clone the repository">
    Clone the project to your local machine:
    
    ```bash
    git clone <repository-url>
    cd it-competition-web
    ```
    
    Replace `<repository-url>` with your actual repository URL.
  </Step>

  <Step title="Install dependencies">
    Install all required packages:
    
    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```

      ```bash pnpm
      pnpm install
      ```

      ```bash bun
      bun install
      ```
    </CodeGroup>
    
    This installs all dependencies from `package.json`, including:
    
    **Core frameworks:**
    - `next@16.1.6` - Next.js framework
    - `react@19.2.3` - React library
    - `react-dom@19.2.3` - React DOM renderer
    
    **Animation & 3D:**
    - `framer-motion@12.29.2` - Animation library
    - `three@0.182.0` - 3D graphics
    - `@react-three/fiber@9.5.0` - React Three.js renderer
    - `@react-three/drei@10.7.7` - Three.js helpers
    - `lenis@1.3.17` - Smooth scrolling
    
    **Styling:**
    - `tailwindcss@4` - Utility-first CSS
    - `class-variance-authority@0.7.1` - Component variants
    - `clsx@2.1.1` - Conditional classes
    - `tailwind-merge@3.5.0` - Merge Tailwind classes
    
    **Additional features:**
    - `maplibre-gl@5.18.0` - Interactive maps
    - `lucide-react@0.563.0` - Icon library
    - `next-themes@0.4.6` - Theme management
  </Step>

  <Step title="Verify installation">
    Check that all dependencies installed correctly:
    
    ```bash
    npm list --depth=0
    ```
    
    You should see all packages listed without errors.
  </Step>

  <Step title="Start development server">
    Launch the development server:
    
    ```bash
    npm run dev
    ```
    
    The server will start on [http://localhost:3000](http://localhost:3000).
  </Step>
</Steps>

## Configuration

### Next.js configuration

The project uses custom Next.js configuration in `next.config.ts`:

```typescript next.config.ts
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  compress: true,
  reactStrictMode: true,
  reactCompiler: true,
  experimental: {
    optimizePackageImports: ['framer-motion', 'lucide-react'],
  },
};

export default nextConfig;
```

**Key settings:**
- `compress: true` - Enables gzip compression
- `reactStrictMode: true` - Enables strict mode for better error detection
- `reactCompiler: true` - Enables experimental React Compiler for optimizations
- `optimizePackageImports` - Optimizes imports from Framer Motion and Lucide React

### Environment variables

The project uses environment variables for configuration. The site URL is set via:

```bash .env.local
NEXT_PUBLIC_SITE_URL=https://iofest.com
```

For local development, this defaults to `http://localhost:3000` if not set.

<Note>
  Environment variables prefixed with `NEXT_PUBLIC_` are exposed to the browser. Keep sensitive data in server-only variables.
</Note>

### Custom fonts

The project uses two custom font families:

1. **Raela Grotesque** - Used for headings and display text
2. **Plus Jakarta Sans** - Used for body text

These are loaded as local fonts in `src/app/layout.tsx` with font weights from 100-900.

```
│   ├── app/                    # Next.js App Router pages

├── tsconfig.json             # TypeScript configuration
└── package.json              # Dependencies and scripts
```

## Development workflow

### Available scripts

| Script | Command | Description |
|--------|---------|-------------|
| **Development** | `npm run dev` | Start dev server with hot reload |
| **Build** | `npm run build` | Create production build |
| **Production** | `npm run start` | Start production server |
| **Lint** | `npm run lint` | Run ESLint code checking |

### Making changes

1. **Edit pages**: Modify files in `src/app/` for page content
2. **Edit components**: Update files in `src/components/`
3. **Edit styles**: Modify `src/app/globals.css` or Tailwind classes
4. **See changes live**: The dev server auto-reloads with your changes

### Building for production

Create an optimized production build:

```bash
npm run build
```

This generates a `.next` directory with:
- Optimized JavaScript bundles
- Static HTML pages
- Optimized images and assets
- Server-side rendering setup

Test the production build locally:

```bash
npm run start
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Installation fails with permission errors">
    On macOS/Linux, you may need to use sudo:
    ```bash
    sudo npm install
    ```
    
    Or fix npm permissions:
    ```bash
    mkdir ~/.npm-global
    npm config set prefix '~/.npm-global'
    export PATH=~/.npm-global/bin:$PATH
    ```
  </Accordion>

  <Accordion title="Node version is too old">
    If you have an older Node.js version:
    
    1. Install [nvm](https://github.com/nvm-sh/nvm) (Node Version Manager)
    2. Install the correct version:
       ```bash
       nvm install 20
       nvm use 20
       ```
  </Accordion>

  <Accordion title="Dependency conflicts">
    If you see peer dependency warnings:
    
    1. Try using `--legacy-peer-deps`:
       ```bash
       npm install --legacy-peer-deps
       ```
    
    2. Or use `--force` (less recommended):
       ```bash
       npm install --force
       ```
  </Accordion>

  <Accordion title="TypeScript errors during build">
    If you encounter TypeScript errors:
    
    1. Ensure TypeScript is installed:
       ```bash
       npm install --save-dev typescript @types/react @types/node
       ```
    
    2. Check `tsconfig.json` is present
    
    3. Run type checking:
       ```bash
       npx tsc --noEmit
       ```
  </Accordion>

  <Accordion title="Out of memory errors">
    If builds fail with memory errors, increase Node.js memory:
    
    ```bash
    NODE_OPTIONS="--max-old-space-size=4096" npm run build
    ```
  </Accordion>

  <Accordion title="WebGL/Three.js issues">
    3D features require WebGL 2. Check:
    
    1. Visit [get.webgl.org](https://get.webgl.org/webgl2/)
    2. Enable hardware acceleration in your browser
    3. Update graphics drivers
    4. Try a different browser (Chrome recommended)
  </Accordion>
</AccordionGroup>
